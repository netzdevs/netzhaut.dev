About

This is an informal guide for using the selfmake Language.

Scripts

Scripts must be submitted using _addFile_ or _addData_ and executed using _run_.The normative way to name a script-file is using the .sm extension. Scripts are executed in the order they are submitted. Variables are shared among scripts. Scripts are made of Functions and Options.

Language

Types

There are no numeric types, only identifiers and strings.

Elements

Functions

Functions that are not inside an Option will be executed once during Script Execution. The selfmake Language provides Standard Functions which are based on build-tool specific needs. For defining your own functions, use _setFunctionCallback_.
copy("foo" "bar")
build(foo)
set(name, "jeff")

Options

Options are executed during Arguments Execution by passing Option matching arguments to _run_. 
"print hello world" --hello { message("hello world!") }
-h { message("hello world!") }
-h pluto { message("hello pluto!") }

Variables

Variables are set and changed using the set() function. They can be used in strings via ${VARIABLE}. There are a number of Standard Variables.

Script Example

set(PROJ_DIR,"${WRK_DIR}")
set(LIB_DEST,"${WRK_DIR}/lib/")
set(BIN_DEST,"${WRK_DIR}/bin/")

lib(lib1,"src/lib1")
lib(lib2,"src/lib2")
lib(lib3,"src/lib2")

bin(bin1,"src/bin1")

compile(bin1,"-std=gnu99 -Wl,-rpath=/usr/local/lib")
link(lib2,"-L${LIB_DEST} -llib1")

source(
  lib1
  "file1.c"
  "file2.c"
  "file3.c"
  lib2
  "file1.c"
  "file2.c"
  "file3.c"
)

"initialize project"
--init
{
    mkdir("external/downloads")
    chdir("external/downloads")

    system("wget -O tmp.zip https://example.com/example.zip")
    system("unzip -o tmp.zip")
}

"push new version"
--push
{
    chdir()
    system("git add .")
    system("git commit -m \"${PROJ_REV}\"")
    system("git push origin HEAD")
}
