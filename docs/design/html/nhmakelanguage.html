Language Specification

This is the language specification for the nhmake Language.

The nhmake Language is a script-language specifically designed for build-tool related tasks. Build-tool related tasks include building, installing and configuring project specific components such as source-code and other data.

Abstract

Scripts written in the nhmake Language are made up of Function, Option and Conditional elements. Function elements can be executed and create diverse functionality. Standard Functions are Function elements and provide default functionality mainly focused on build-tool related tasks. Option elements are made up of Function elements and can be executed dynamically by passing arguments to _run_. Conditional elements are TODO.

Variables can be set and accessed using special notation inside strings. Standard Variables allow control over specific runtime parameters.

Scripts

A script can be either a script-file or just script-data. The normative way to name a script-file is using the .sm extension. Scripts are added to a Runtime using _addFile_ or _addData_ and are executed using _run_. Scripts are made up of Language Elements.

Language Elements

Function

A Function element is identified by an identifier and uses round parentheses to enclose arguments. A function can have any number of identifier or string arguments. Commas between arguments are optional.

Examples

identifier()
identifier(identifier, string)
identifier(string identifier)
identifier(string, string string)

Semantics

Function elements are executed during Script Execution or during Arguments Execution. When Function Callback Execution does not apply, a Function element must match a Standard Function.

Option

Option elements are identified by an identifier and can be prefixed with a string for description purposes. An option is either a long-option declared using a -- identifier-prefix or a short-option declared using a - identfier-prefix. An option block can have any number of Function elements. Between option identifier and option block can be any number of arguments in the form of additional identifiers.

Examples

--identifier {}
-identifier identifier {}
string -identifier {}
string --identifier identifier identifier {Function}
--identifier {Function Function}

Semantics

Option elements are executed during Arguments Execution.

Variables

A variable is declared and changed using the set() standard function.

String Preprocessing

Function arguments in the form of strings are preprocessed by expanding variable references. A variable can be referenced by using the reference notation ${EXAMPLE}. A full string using a variable reference could look like "My name is ${NAME}" and would possibly be replaced after preprocessing to "My name is Mud".

Standard Variables

Some variables are builtin and control the behavior of the runtime. They can be changed just as any other variable. These variables are known as Standard Variables.
